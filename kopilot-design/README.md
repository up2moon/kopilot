## 1. 시장 현황

Kopilot은 마이데이터 기반 소비 분석 시장과 20~30대의 소비·자산관리 니즈가 동시에 커지는 흐름을 배경으로 한다. 사용자는 이미 금융 데이터를 한 앱에서 모아 보는 경험에 익숙해졌지만, 시장의 다수 서비스는 아직 `조회 → 분석 → 상품 추천` 중심에 머문다. Kopilot은 이 데이터를 `예산 설정 → AI 절약 코칭 → 챌린지 → 투자효과 시뮬레이션`으로 연결해 실제 행동 변화를 만드는 것을 목표로 한다.

## 2. 문제점 분석

① 소비 데이터는 많지만 행동 변화로 이어지기 어렵다.

- 기존 금융·가계부 앱은 계좌, 카드, 대출, 투자 내역을 모아 보여주고 소비를 자동 분류하는 데 강점이 있다.
- 그러나 사용자가 다음에 무엇을 줄여야 하는지, 이번 주에 어떤 행동을 해야 하는지까지 이어지는 실행 루프는 약하다.
- 근거: 토스는 공식 소개에서 계좌·카드 연결 후 잔액, 대출·투자 내역, 일자별 소비와 수입을 한 번에 보는 기능을 강조한다. 뱅크샐러드는 자동 가계부, 예산 설정, 소비 패턴 기반 카드 추천을 제공한다.
- Kopilot 대응: 대시보드에서 소비 요약을 보여준 뒤 `AI 코치와 대화하기`, `소비 상세 보기`, `챌린지 수행 인증하기`로 다음 행동을 바로 연결한다.

② 절약 목표를 처음부터 개인화하기 어렵다.

- 사용자가 직접 가계부를 꾸준히 입력하거나 예산을 수동 설정해야 하면 초기 이탈 가능성이 높다.
- 마이데이터를 연동해도 단순 조회에 그치면 사용자가 자신의 관리 카테고리와 월 예산을 명확히 설정하지 못한다.
- 근거: 2024년 전국민 금융이해력 조사에서 20대 청년층은 금융행위 중심으로 상대적으로 취약하게 나타났다. 이는 소비 관리 서비스를 단순 정보 제공이 아니라 안내형 설정 플로우로 설계해야 함을 시사한다.
- Kopilot 대응: 첫 로그인 초기 설정에서 `환영 → 마이데이터 연동 → 카테고리 선택 → 예산 목표 설정 → 완료`의 6단계 온보딩을 제공하고, 식비·카페·쇼핑을 기본 선택값으로 제안한다.

③ 절약의 보상이 즉시 체감되지 않는다.

- 절약은 당장의 불편을 요구하지만, 사용자가 체감하는 보상은 늦게 나타난다.
- 기존 자산관리 서비스는 순자산과 소비 현황을 보여주거나 금융상품 혜택을 비교하는 데 초점이 있어, 오늘 아낀 금액이 미래 자산 가치로 어떻게 바뀔 수 있는지 직관적으로 보여주는 경험은 제한적이다.
- 근거: 카카오페이는 공식 소개에서 송금·결제·멤버십, 자산관리, 문서함, 청구서, 투자·대출·보험 등 금융 편의 기능을 강조한다. 뱅크샐러드와 토스도 금융상품 비교·추천 기능을 핵심 축으로 운영한다.
- Kopilot 대응: 투자효과 화면에서 `이번 달 커피 절약액 60,000원`을 스타벅스, S&P500 ETF, 정기예금 등과 비교하는 참고용 시뮬레이션으로 보여주고, 투자 권유가 아님을 명확히 고지한다.

왜 기존 서비스로는 충분하지 않은가?

기존 서비스는 금융 데이터를 모아 보여주고 상품 선택을 돕는 데 강하지만, 사용자가 `무엇을 줄이고`, `이번 주에 무엇을 실천하며`, `그 절약이 어떤 미래 가치로 이어지는지`를 하나의 흐름에서 체감하게 만드는 구조는 약하다. Kopilot은 첫 로그인부터 사용자의 관리 카테고리와 예산을 수집하고, 대시보드·AI 챗봇·챌린지·랭킹·투자효과를 연결해 소비 관리가 반복 행동으로 이어지도록 설계한다.

Ⅱ. 서비스 소개

## 1. 서비스 개요

Kopilot은 소비 관리가 어렵거나 자산 형성을 막 시작한 사용자가 마이데이터 기반 소비 분석을 바탕으로 절약 목표를 설정하고, AI 코치의 추천을 받아 매일 실천 가능한 절약 행동을 이어갈 수 있게 돕는 모바일 금융 코파일럿 서비스다.

서비스의 핵심 경험은 `소비 확인`에서 끝나지 않고 `예산 설정 → AI 절약 코칭 → 챌린지 수행 → 절약액 시뮬레이션`으로 이어지는 실행 흐름이다. 사용자는 첫 로그인 온보딩에서 마이데이터 연동 여부를 선택하고, 관리할 소비 카테고리와 월 예산을 설정한다. 이후 대시보드에서 이번 달 소비, 절약 가능 금액, AI 코치 추천, 챌린지, 익명 랭킹, 투자효과 시뮬레이션을 확인한다.

| 한 문장 정의 | 소비를 줄이는 순간을 자산 형성 행동으로 연결하는 AI 절약 코치 |
| 주요 사용자 | 20~30대 사회초년생, 소비 통제가 어려운 월급 생활자, 가계부를 꾸준히 쓰지 못하는 사용자 |

| 주요 화면 | 로그인, 회원가입, 첫 로그인 초기 설정, 대시보드, 내 소비 상세, AI 절약 챗봇, 랭킹, 챌린지, 투자효과, 마이 |

### MVP 우선순위

1. 첫 로그인 온보딩과 예산 설정을 가장 먼저 구현한다.
2. 대시보드에서 소비 요약과 AI 코치 진입점을 구현한다.
3. 챌린지와 투자효과는 mock data 기반으로 동작하는 최소 기능부터 구현한다.
4. 실제 마이데이터 API와 생성형 AI 연동은 이후 확장 범위로 둔다.

### Figma 기준 화면 반영

- 첫 로그인 초기 설정은 `환영`, `마이데이터 안내`, `연동 처리`, `카테고리 선택`, `예산 목표 설정`, `설정 완료` 6개 화면 흐름을 따른다.
- 대시보드는 `이번 달 소비 1,245,000원`, `지난달보다 8.3% 증가`, `AI 코치와 대화하기`를 핵심 정보로 배치한다.
- 주요 앱 화면은 하단 탭 `홈`, `랭킹`, `챌린지`, `투자효과`, `마이` 구조를 따른다.
- AI 챗봇과 투자효과 화면에는 금융 조언 또는 투자 권유가 아니라 참고용 정보라는 고지를 표시한다.
