# Kopilot PRD

## 1. 제품 개요

**Kopilot(코파일럿)**은 사용자의 절약과 자산 형성을 함께하는 AI 금융 코파일럿이다. 마이데이터 기반 소비 분석, 생성형 AI 절약 코칭, 절약 챌린지, 익명 랭킹, 미래 자산 가치 시뮬레이션을 통해 사용자가 소비 습관을 개선하고 절약을 실제 자산 형성 행동으로 연결하도록 돕는다.

MVP는 모바일 웹 화면을 우선한다. 현재 와이어프레임 기준 화면 크기는 기본 `390 x 844`이며, 스크롤이 필요한 화면은 너비 `390px`을 유지하고 콘텐츠 길이에 맞춰 높이를 확장한다.

## 3. 기획 배경과 문제 정의

마이데이터 확산으로 개인 금융 데이터를 통합 활용할 수 있는 환경이 마련되었다. 동시에 20~30대 사회초년생을 중심으로 소비 관리와 자산 형성에 대한 관심이 증가하고 있다. 그러나 기존 금융 서비스는 소비 내역 기록과 분석에 머무르는 경우가 많아 사용자의 행동 변화를 지속적으로 이끌어내는 데 한계가 있다.

Kopilot은 소비 기록을 분석, 코칭, 챌린지, 시뮬레이션으로 연결해 사용자가 실천 가능한 절약 행동을 반복하도록 만든다.

주요 문제는 다음과 같다.

- 기존 가계부 서비스는 기록 중심이라 사용자 참여와 지속성이 낮다.
- 절약은 단기적으로 불편하게 느껴져 쉽게 포기하게 된다.
- 소비 습관을 지속적으로 개선할 동기부여와 보상 체계가 부족하다.
- 소비를 절약과 자산 형성으로 연결해 체감하게 만드는 서비스가 부족하다.

## 4. 대상 고객

핵심 사용자는 20~30대 사회초년생, 월급은 받지만 소비 통제가 어려운 사용자, 가계부를 꾸준히 사용하지 못하는 사용자, 저축 및 자산 형성을 시작하려는 사용자다.

사용자 니즈는 다음과 같다.

- 내 소비가 어디에서 새고 있는지 쉽게 알고 싶다.
- 무리하지 않고 실천 가능한 절약 방법을 추천받고 싶다.
- 절약이 미래 자산 형성에 어떤 영향을 주는지 직관적으로 보고 싶다.
- 절약을 꾸준히 이어갈 수 있는 동기와 보상을 원한다.

## 5. 제품 목표와 성공 지표

제품 목표는 다음과 같다.

- 사용자의 소비 패턴을 자동 분석하고 핵심 낭비 영역을 도출한다.
- 첫 로그인 단계에서 마이데이터 연동, 관심 카테고리, 월 예산 목표를 수집한다.
- 개인 맞춤형 절약 전략과 챌린지를 제공한다.
- 절약 금액을 미래 자산 가치로 시각화해 행동 변화를 유도한다.
- 보상, 익명 랭킹, 배지를 통해 반복 사용과 장기 참여를 높인다.

성공 지표는 다음과 같다.

- 회원가입 완료율
- 첫 로그인 온보딩 완료율
- 마이데이터 연동 완료율 또는 건너뛰기 후 예산 설정 완료율
- 주간 활성 사용자 수(WAU)
- 소비 상세 화면 진입률
- AI 코치 대화 시작률 및 추천 수락률
- 챌린지 시작률 및 완료율
- 월평균 절약 금액
- 4주 이상 유지율
- 프리미엄 구독 전환율

대시보드 이후 주요 앱 화면에는 하단 메뉴바와 플로팅 AI 챗봇 버튼을 둔다. 하단 탭은 `홈`, `랭킹`, `챌린지`, `투자효과`, `마이` 5개로 구성한다.

## 7. 핵심 기능 요구사항

### 7.1 인증

로그인 화면은 Kopilot의 핵심 가치와 로그인 폼을 함께 보여준다.

- 히어로 문구: `소비를 줄이는 순간, 자산 형성이 시작돼요`
- 보조 문구: `마이데이터 소비 분석과 절약 챌린지를 한 화면에서 확인하세요.`
- 인사이트 프리뷰: `이번 달 소비 인사이트`, `커피 지출 300,000원`, `+60,000원 절약 가능`
- 입력 필드: 이메일, 비밀번호
- 주요 CTA: `로그인`
- 회원가입 이동 링크: `무료로 시작하기`

회원가입 화면은 3분 내 시작 가능한 절약 코치 경험을 강조한다.

- 히어로 문구: `3분 만에 시작하는 나만의 절약 코치`
- 입력 필드: 이름, 이메일, 비밀번호
- 필수 약관 동의 체크박스
- 주요 CTA: `가입하고 분석 시작`
- 로그인 이동 링크 제공

### 7.2 첫 로그인 초기 설정

첫 로그인 초기 설정은 사용자가 처음 로그인했을 때 등장하는 6단계 온보딩 플로우다. 목적은 마이데이터 연동을 유도하고, 관리할 소비 카테고리와 카테고리별 월 예산 목표 금액을 수집하는 것이다.

진입 조건은 다음 중 하나 이상을 만족하는 경우다.

- 첫 로그인 사용자
- 마이데이터 연동을 아직 완료하지 않은 사용자
- 카테고리별 예산 설정을 아직 완료하지 않은 사용자

플로우는 다음 순서로 진행한다.

```text
Step 1. 환영
→ Step 2. 마이데이터 연동 안내
→ Step 3. 마이데이터 연동 처리 또는 건너뛰기
→ Step 4. 소비 카테고리 선택
→ Step 5. 카테고리별 목표 예산 설정
→ Step 6. 설정 완료
→ /dashboard
```

Step 1 환영 화면 요구사항:

- 문구: `절약과 자산 형성을 함께하는 AI 코파일럿, KoPilot`
- 제공 가치: 소비 분석, AI 절약 코치, 절약 챌린지
- 투자효과는 절약액의 참고용 시뮬레이션임을 하단에 고지
- CTA: `시작하기`

Step 2 마이데이터 안내 화면 요구사항:

- 문구: `소비 분석을 위해 마이데이터 연동이 필요해요`
- 안내 항목: 결제 내역 분석, 소비 카테고리 분류, 사용자 동의 기반
- 실제 금융 데이터는 사용자 동의 후에만 연결된다고 명시
- CTA: `마이데이터 연동하기`
- 보조 CTA: `나중에 하기`

Step 3 마이데이터 연동 처리 요구사항:

- 카드 결제내역, 선불/간편결제 내역, 은행 입출금 내역을 연결 대상으로 표시
- 실제 API가 없는 MVP에서는 연동 버튼 클릭 시 mock 연동 완료 상태로 처리
- 연동 완료 시 `myDataConnected=true` 저장
- 건너뛰기 시 mock data 기반으로 카테고리 선택 단계 진입

Step 4 카테고리 선택 요구사항:

- 문구: `관리하고 싶은 소비 카테고리를 선택해주세요`
- 선택한 카테고리를 기준으로 월 예산과 절약 챌린지를 설정할 수 있음을 설명
- 기본 선택값: 식비, 카페·간식, 쇼핑
- 선택 가능 카테고리: 식비, 카페·간식, 쇼핑, 배달, 교통, 구독, 문화, 통신
- CTA: `다음`
- 보조 CTA: `이전`

Step 5 예산 목표 설정 요구사항:

- 문구: `이번 달 예산을 서로를 위해 지키고`
- 월 정보와 총 남은 금액 표시: 예시 `2026년 5월`, `총 220,000원 남음`
- 추천 예산 카드 표시: 최근 3개월 평균 소비를 기준으로 절감 목표 제안
- 카테고리별 표시 정보: 아이콘, 카테고리명, 월 목표 금액, 현재 사용 금액, 남은 금액, 진행률
- 입력 방식: 직접 입력, 금액 증감 버튼, 추천 예산 자동 적용
- CTA: `예산 설정 완료`
- 보조 CTA: `추천 예산 적용`

Step 6 설정 완료 요구사항:

- 문구: `예산 설정이 완료되었어요`
- 초기 설정 요약: 마이데이터 상태, 월 예산 설정 개수
- 다음 행동 안내: 대시보드에서 소비 분석과 AI 절약 코치를 확인
- CTA: `대시보드로 이동`
- 완료 시 `firstLoginCompleted=true`, `budgetSetupCompleted=true` 저장

### 7.3 대시보드

대시보드는 로그인 이후 기본 홈 화면이다. 마이데이터 기반 소비 현황과 AI 절약 코치 진입점을 한 화면에서 제공한다.

- 상단 브랜드와 프로필 pill 표시
- 인사말: `진원님, 이번 달 소비를 확인해 보세요.`
- 보조 문구: `마이데이터 기반으로 새는 지출을 정리했어요.`
- 소비 요약 카드: 이번 달 소비 금액, 지난달 대비 증감률, 총 결제 건수
- 예시 데이터: `1,245,000원`, `지난달보다 8.3% 증가`, `42건`
- `소비 상세 보기` CTA로 내 소비 상세 화면 이동
- AI 절약 코치 카드: 절약 가능 금액과 추천 메시지 표시
- `AI 코치와 대화하기` CTA로 챗봇 화면 이동
- 플로팅 AI 챗봇 버튼 제공

### 7.4 내 소비 상세

내 소비 상세 화면은 대시보드의 요약 정보를 확장해 기간별, 카테고리별, 일자별 소비를 확인하는 화면이다.

- 상단 뒤로가기 버튼과 제목 `내 소비 상세`
- 기간 필터: 이번 달, 최근 3개월, 카테고리
- 총소비 카드: 총소비, 지난달 대비 증감률, 결제 건수, 평균 결제액
- 카테고리별 상세: 식비, 카페, 교통 등 주요 카테고리의 금액과 전체 비중
- 일자별 소비 차트: 주차별 또는 일자별 막대 차트
- 결제 내역 목록: 가맹점명, 카테고리, 결제 일시, 금액
- 예시 결제: 스타벅스 부산대점, 배달의민족, 카카오택시

### 7.5 AI 절약 챗봇

AI 절약 챗봇은 소비 패턴을 바탕으로 사용자가 바로 실행할 수 있는 절약 방법을 대화형으로 제안한다.

- 상단 상태: `분석 완료`
- 인트로 카드: `커피와 배달 지출을 줄이면 이번 달 94,000원을 아낄 수 있어요.`
- 대화 예시: 최근 카페 결제 빈도 분석, 이번 주 절약 방법 제안
- 추천 질문: 이번 주 미션 만들기, 구독 지출 줄이기, 배달비 분석하기
- 채팅 입력 필드와 전송 버튼 제공
- 하단 고지: AI 코칭은 참고용이며 투자 권유가 아님

### 7.6 익명 랭킹

랭킹 화면은 절약 행동을 게임화해 지속 참여를 유도한다.

- 제목: `익명 랭킹`
- 설명: 자동 닉네임과 프로필 이미지로 이번 달 절약 랭킹 확인
- 내 랭킹 카드: 익명 닉네임, 이번 달 예상 절약액, 순위
- 전체 랭킹 목록: 상위 사용자 닉네임과 절약액
- 개인정보 보호를 위해 실명 대신 자동 생성 닉네임을 사용

### 7.7 챌린지

챌린지 화면은 AI가 추천하거나 랜덤 배정한 절약 미션을 수행하는 화면이다.

- 제목: `챌린지`
- 설명: `AI가 오늘 수행할 절약 미션을 랜덤으로 배정해요.`
- 오늘의 랜덤 미션 카드: 난이도, 미션명, 예상 절약액
- 예시 미션: `평일 커피 1회 쉬기`, `택시 대신 지하철`, `배달비 줄이기`, `구독 확인하기`
- CTA: `수행 인증하기`
- 이번 주 미션 목록과 상태 표시: 완료, 진행중 등

### 7.8 투자효과

투자효과 화면은 절약한 금액의 기회비용과 미래 가치를 참고용으로 보여준다.

- 제목: `투자효과`
- 설명: `아낀 돈의 기회비용을 참고용 시뮬레이션으로 확인해요.`
- 절약액 카드: 예시 `이번 달 커피 절약액 60,000원`
- 비교 문구: `이 돈을 스타벅스에 투자했다면 현재 약 68,400원이 되었어요.`
- 비교 기준: 스타벅스, S&P500 ETF, 정기예금
- 모든 화면에 투자 권유가 아닌 참고용 시뮬레이션이라는 고지를 포함

### 7.9 마이

마이 화면은 계정과 서비스 설정을 관리하는 화면이다.

- 프로필 카드: 사용자 이름, 절약 레벨, 배지 수
- 설정 메뉴: 마이데이터 연결 관리, 알림 설정, 로그아웃
- 하단 탭에서 마이 활성 상태 표시

## 10. 상태와 예외 처리

### 10.1 온보딩 상태

| 상태 | 설명 |
| --- | --- |
| `NOT_STARTED` | 첫 로그인 설정 시작 전 |

| `MYDATA_CONNECTING` | 마이데이터 연동 진행 중 |
| `MYDATA_CONNECTED` | 마이데이터 연동 완료 |
| `CATEGORY_SELECTING` | 카테고리 선택 중 |
| `BUDGET_SETTING` | 예산 설정 중 |
| `COMPLETED` | 초기 설정 완료 |

### 10.2 예외 상황

마이데이터 연동 실패:

- 문구: `마이데이터 연동에 실패했어요. 잠시 후 다시 시도하거나, 기본 설정으로 시작할 수 있어요.`
- CTA: 다시 시도
- 보조 CTA: 기본 설정으로 시작

예산 미입력:

- 문구: `선택한 카테고리의 예산을 모두 입력해주세요.`
- 예산 설정 완료 버튼을 비활성화하거나 에러 메시지를 표시한다.

예산 금액이 0원 이하:

- 문구: `예산은 1원 이상으로 설정해주세요.`

AI 코치 응답 실패:

- 네트워크 오류 또는 생성 실패 시 재시도 버튼을 제공한다.
- 실패해도 기존 대시보드, 챌린지, 예산 데이터는 유지한다.

## 11. 디자인 요구사항

`DESIGN.md`를 상위 디자인 기준으로 삼는다.

- 전체 분위기는 토스처럼 간결하고 신뢰감 있게 유지한다.
- 주요 색상은 `#3182F6`, 배경은 `#F7F9FC`, 긍정 수치 강조는 `#00A661`을 사용한다.
- 보조 텍스트는 `#6B7684`, 플레이스홀더는 `#8B95A1`을 기준으로 한다.
- 폰트는 `Noto Sans KR`과 `Inter` 조합을 우선한다.
- 버튼과 입력 필드는 높이 `58px`, 라운드 `14~16px`을 기준으로 한다.
- 카드형 정보 영역은 흰색 배경, `20~22px` 라운드, 부드러운 그림자를 사용한다.
- 모바일 좌우 기본 여백은 `24px`을 우선한다.
- 문구는 짧고 행동 중심으로 작성한다.
- 투자 관련 표현은 투자 권유가 아니라 참고용 시뮬레이션임을 명확히 한다.

- 카테고리 선택과 월 예산 설정
- 대시보드 소비 요약
- 내 소비 상세

- 익명 랭킹
- 투자효과 참고용 시뮬레이션
- 마이 페이지 기본 설정 메뉴

- 친구 랭킹 및 그룹 챌린지

- 금융상품 연계
- 프리미엄 AI 코칭
- 맞춤형 쿠폰 및 제휴 리워드
- 장기 자산 목표 관리

## 13. 정책 및 유의사항

- 마이데이터 연동 전 사용자 동의를 명확히 받아야 한다.
- 소비 데이터는 최소 수집 및 목적 제한 원칙에 따라 처리한다.
- 익명 랭킹에서는 실명과 직접 식별 가능한 개인정보를 노출하지 않는다.
- AI 코칭은 금융 조언이 아니라 소비 관리 참고 정보로 제공한다.
- 투자 시뮬레이션은 투자 권유가 아니며 과거 수익률이 미래 수익을 보장하지 않는다는 고지를 포함한다.
- 금융상품 추천 또는 중개 기능이 포함될 경우 관련 금융 규제 검토가 필요하다.

## 14. 완료 기준

다음 조건을 만족하면 MVP 구현 완료로 본다.

- 로그인과 회원가입 화면이 와이어프레임의 정보 구조와 주요 문구를 따른다.
- 로그인 성공 후 `firstLoginCompleted` 값에 따라 `/first-login` 또는 `/dashboard`로 분기한다.
- 첫 로그인 사용자가 6단계 온보딩을 완료할 수 있다.
- 마이데이터 연동 완료와 건너뛰기 경로가 모두 동작한다.
- 사용자가 관리할 소비 카테고리를 선택할 수 있다.
- 사용자가 선택한 카테고리별 예산 목표 금액을 입력하거나 추천 예산을 적용할 수 있다.
- 설정 완료 후 대시보드로 이동한다.
- 대시보드에서 소비 요약, AI 코치 카드, 소비 상세 이동, 챗봇 이동이 가능하다.
- 내 소비 상세에서 기간 필터, 카테고리별 상세, 일자별 차트, 결제 내역을 확인할 수 있다.
- AI 절약 챗봇에서 추천 질문과 메시지 입력 흐름을 확인할 수 있다.
- 랭킹, 챌린지, 투자효과, 마이 화면이 하단 탭 구조로 이동 가능하다.
- 투자효과와 AI 코칭 화면에 참고용/비권유 고지가 표시된다.
- 실제 API가 없어도 mock data만으로 전체 화면 흐름을 확인할 수 있다.
